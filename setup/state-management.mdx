## Overview

State management in Davia allows you to maintain application state across different tasks and function calls. Davia's state management system provides a clean and intuitive way to work with persistent data using Python's type annotations. By leveraging the `Annotated` type and `State` class, you can easily access, modify, and share state between different tasks without complex boilerplate code.

## Working with State

### Initializing State

To use state in your Davia application, initialize it during app creation:

```python
from davia import Davia, run_server, State
import pandas as pd

# Initialize your state
products = pd.DataFrame(
    {
        "name": ["Laptop", "Phone", "Headphones"],
        "price": [999.99, 699.99, 199.99],
        "quantity": [1, 1, 2],
    }
)

# Create a state dictionary with named keys
state = {"products": products}

# Pass the state to your Davia app
app = Davia(
    title="Product Manager",
    description="Manage your shopping list with ease",
    state=state,
)
```

### Accessing State in Tasks

Davia provides two main approaches for accessing state in your tasks:

#### Method 1: Using Annotated Type with State Key

The most common approach uses Python's `Annotated` type to inject a specific state component directly as a parameter:

```python
from typing import Annotated

@app.task
def add_product(
    products: Annotated[pd.DataFrame, State("products")],  # Access 'products' from state
    name: str,
    price: float,
    quantity: int = 1,
):
    # Modify the state
    new_product = pd.DataFrame(
        {"name": [name], "price": [price], "quantity": [quantity]}
    )
    products.loc[len(products)] = new_product.iloc[0]
```

The `Annotated[type, State("key_name")]` syntax tells Davia to:

1. Locate the value with key "key_name" from the application state
2. Inject that value into your function parameter
3. Expect the value to match the specified type (e.g., `pd.DataFrame`)

#### Method 2: Accessing the Entire State Object

You can also access the complete state object and retrieve components from it:

```python
@app.task
def get_element_from_state(state: State):
    # Access any part of the state using dictionary syntax
    return state["products"]
```

When you annotate a parameter with just `State` (without `Annotated`), Davia injects the entire state dictionary. You can then access any component using regular dictionary syntax.

This approach is particularly useful when you need to:

- Access multiple state components in a single function
- Dynamically determine which state components to use based on runtime conditions
- Handle state components that might not exist

### Reading State

When you only need to read state without modifying it, you can use either method:

Using Method 1 (Annotated with specific state key):

```python
@app.task
def get_products(
    products: Annotated[pd.DataFrame, State("products")],
) -> dict:
    return products.to_dict()
```

Using Method 2 (Accessing complete state):

```python
@app.task
def get_products(state: State) -> dict:
    products = state["products"]
    return products.to_dict()
```

### Modifying State

State modifications are applied directly to the parameter received in your function:

```python
@app.task
def update_price(
    products: Annotated[pd.DataFrame, State("products")],
    product_name: str,
    new_price: float,
):
    products.loc[products["name"] == product_name, "price"] = new_price
```

## Working with Multiple State Components

You can have multiple components in your state and access them separately:

```python
# Initial state with multiple components
state = {
    "products": products_df,
    "user_preferences": user_prefs_dict
}

@app.task
def update_user_theme(
    preferences: Annotated[dict, State("user_preferences")],
    theme: str
):
    preferences["theme"] = theme
```

## Supported State Types

Davia supports various data types for state:

- Pandas DataFrames
- Dictionaries
- Lists
- Custom objects
- Primitive types (within a container)

## Complete Implementation

```python
from davia import Davia, run_server, State
from typing import Annotated
import pandas as pd

# Initialize state
products = pd.DataFrame(
    {
        "name": ["Laptop", "Phone"],
        "price": [999.99, 699.99],
        "quantity": [1, 1],
    }
)

state = {"products": products}

app = Davia(
    title="Product Manager",
    description="Manage your products",
    state=state,
)

# Method 1: Using Annotated with specific state key
@app.task
def add_product(
    products: Annotated[pd.DataFrame, State("products")],
    name: str,
    price: float,
    quantity: int = 1,
):
    new_product = pd.DataFrame(
        {"name": [name], "price": [price], "quantity": [quantity]}
    )
    products.loc[len(products)] = new_product.iloc[0]


@app.task
def get_products(
    products: Annotated[pd.DataFrame, State("products")],
) -> dict:
    return products.to_dict()

if __name__ == "__main__":
    run_server(app, browser=False)
```

## Conclusion

State management in Davia provides a convenient way to maintain application data across tasks. By using type annotations with the `State` class, you can easily access and modify your application's state in a type-safe manner.
